## Matriz de Rastreabilidade


| ID   | Descrição                                                                                      | Implementado | Pré-Rastreabilidade | Artefatos | Elos |
|------|------------------------------------------------------------------------------------------------|--------------|--------------------|-----------|------|
| INT01| O aplicativo deve permitir que os usuários se autentiquem de forma segura, usando credenciais únicas, como CPF e senha. | Sim | - | - | - |
| INT02| Deve ser possível acessar os dados diretamente da secretaria de estado de economia do distrito federal | Sim | - | - | - |
| INT03| O aplicativo deve permitir a recuperação de senha por meio de um processo seguro e validado. | Sim | - | - | - |
| INT04| Os usuários devem poder acessar e visualizar informações sobre créditos acumulados. | Não | - | - | - |
| INT05| O aplicativo deve possibilitar a consulta do histórico de notas fiscais associadas à conta do usuário. | Sim | - | - | - |
| INT06| Permitir que os usuários consultem estabelecimentos comerciais parceiros do programa Nota Legal. | Não | - | - | - |
| INT07| Os usuários devem poder resgatar os créditos acumulados de forma clara e fácil. | Não | - | - | - |
| INT08| Deve ser possível visualizar promoções e descontos oferecidos em estabelecimentos participantes. | Sim | - | - | - |
| INT09| O aplicativo deve ter uma interface intuitiva, fácil de navegar e que proporcione uma boa experiência ao usuário. | Sim | - | - | - |
| INT10| Deve ser oferecido suporte para diferentes tamanhos de tela e dispositivos móveis. | Não | - | - | - |
| INT11| Deve ser disponibilizada uma forma de entrar em contato com a Secretária de Estado de Economia no caso de algum problema específico do usuário | Sim | - | - | - |
| INT20| Garantir que o usuário consiga realizar login com a conta do GOV | Não | - | - | - |
| INT21| Possuir informações sobre desbloqueio de valores bloqueados | Não | - | - | - |
| INT22| Possuir informações sobre o percentual de concessão de crédito de acordo com o tipo de estabelecimento que emitiu a nota fiscal | Não | - | - | - |
| INT23| Possibilitar que o usuário aumente a fonte | Não | - | - | - |
| INT24| Possuir um modo escuro e um modo claro | Não | - | - | - |
| INT25| Salvar os dados bancários da conta usada na indicação | Não | - | - | - |
| INT26| Deve ser possível encontrar o recibo da indicação | Não | - | - | - |
| INT30| O usuário deve conseguir emitir a segunda via da dívida ativa | Não | - | - | - |
| INT12| Garantir a segurança das informações dos usuários durante a transmissão e armazenamento de dados. | Sim | - | - | - |
| INT13| Assegurar conformidade com regulamentações de proteção de dados vigentes no Brasil. | Sim | - | - | - |
| INT14| Garantir uma resposta rápida e eficiente, minimizando o tempo de carregamento das páginas. | Não | - | - | - |
| INT15| Manter a disponibilidade do aplicativo, minimizando tempo de inatividade para garantir uma experiência contínua. | Não | - | - | - |
| INT16| Garantir uma interface de usuário amigável, facilitando a interação e a compreensão das funcionalidades do aplicativo. | Sim | - | - | - |
| INT17| Certificar-se de que o aplicativo é compatível com uma ampla variedade de dispositivos móveis e sistemas operacionais. | Não | - | - | - |
| INT18| Garantir que o aplicativo seja otimizado para diferentes navegadores web. | Não | - | - | - |
| INT19| Garantir que ao fechar o aplicativo o usuário seja deslogado | Não | - | - | - |
| INT27| O usuário não deve conseguir colocar uma quantidade diferente de 7 caracteres ao cadastrar a placa de um veículo | Não | - | - | - |
| INT28| O usuário não deve conseguir colocar uma quantidade diferente de 11 números ao cadastrar o renavan de um veículo | Não | - | - | - |
| INT29| O usuário não deve conseguir colocar uma quantidade diferente de 8 números ao cadastrar a inscrição de um imóvel | Não | - | - | - |
